# 은행·페이 공식 로고 적용 가이드

## 현재 상태

로고를 지정하지 않은 항목은 **브랜드 컬러 워드마크**(예: 노란 배경에 `B`, 파란 배경에 `toss`)로 표시됩니다.
공식 로고 이미지를 넣으면 워드마크 대신 실제 로고가 표시됩니다.

## 왜 로고가 기본 포함되어 있지 않은가

은행·페이 로고는 각 사의 **등록상표**입니다. 제3자 서비스가 무단으로 로고를 복제·재배포하는 것은
상표권 침해에 해당할 수 있고, 각 사 브랜드 가이드라인에서 사전 승인을 요구하는 경우가 많습니다.
따라서 로고 파일은 저장소에 포함하지 않고, **운영자가 직접 확보·배치하는 구조**로 만들었습니다.

## 적용 방법

### 1. 로고 파일 확보

각 사 공식 채널에서 받으세요.

| 확보 경로 | 비고 |
|---|---|
| 각 사 홈페이지 → "브랜드/BI 소개" 또는 "홍보자료실" | 가장 확실. 사용 조건 함께 확인 |
| 각 사 제휴·마케팅 담당 문의 | 가맹점·제휴사 대상 로고 키트를 제공하는 경우 있음 |
| App Store / Google Play 앱 아이콘 | 앱 식별 목적의 지명적 사용(nominative use)에 해당. 다만 각 사 정책 확인 권장 |

**[검증필요]** 상표 사용 가능 범위는 각 사 브랜드 가이드라인과 개별 정책에 따라 다릅니다.
실제 상업적 운영 전에 사용하려는 각 사의 가이드라인을 확인하거나 담당자에게 문의하세요.

### 2. 파일 배치

이 폴더(`assets/banks/`)에 넣습니다.

```
assets/banks/
  kakaobank.png
  tossbank.png
  kbank.png
  kb.png
  ...
```

권장 규격: **정사각형 PNG, 배경 투명 또는 흰색, 최소 68x68px** (레티나 대응 위해 2배수)

### 3. 코드에 연결

`index.html`의 `PAYS` / `BANKS` 배열에서 해당 항목에 `logo` 필드를 추가합니다.

```javascript
// 변경 전 (워드마크 표시)
{ n: '카카오뱅크', c: '카카오뱅크', bg: '#FFCC00', fg: '#3C1E1E', i: 'B', s: 'kakaobank://', appId: '1258016944' },

// 변경 후 (공식 로고 표시)
{ n: '카카오뱅크', c: '카카오뱅크', bg: '#FFCC00', fg: '#3C1E1E', i: 'B', s: 'kakaobank://', appId: '1258016944',
  logo: 'assets/banks/kakaobank.png' },
```

`bg` / `fg` / `i` 필드는 **지우지 마세요**. 로고 파일 로드에 실패하면 자동으로 워드마크로 폴백합니다.

### 4. 확인

- 로고가 정상 표시되는지
- 파일명을 일부러 틀리게 해봤을 때 워드마크로 폴백되는지 (버튼이 깨지지 않아야 함)

## 부분 적용 가능

전체를 한 번에 할 필요 없습니다. 고객이 가장 많이 쓰는 상위 5~6개(카카오뱅크·토스·KB·농협·신한·카카오페이)만
먼저 넣어도 체감 완성도가 크게 올라갑니다. 나머지는 워드마크로 남겨두면 됩니다.
